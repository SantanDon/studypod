StudyPodLM: Single Host Voice Integration Map

🎙️ Current Architecture Active

  • Script Engine: Generates two-host dialogues using alternating presets (Alex & Sarah).
  • TTS Synthesis: Stitches individual audio blobs segment-by-segment with static pauses.
  • UI Panel: Fixed dual host voice parameters inside the Studio sidebar controls.

⚡ Single Speaker Variety Proposed

  • Script Engine: Updates prompt to generate single-voice lectures or audiobook formats.
  • TTS Synthesis: Feeds the entire script continuously to a single selected host voice.
  • UI Panel: Adds a toggle switch between Conversational Podcast and Solo Audiobook.

💡 Rationale & Benefits

Switching to a single speaker halves TTS processing latency by eliminating multi-voice context switches, reduces audio stitching artifacts, and provides academic variety for users who prefer structured solo lectures over conversational back-and-forth.

🛠️ Implementation Steps

  1. Modify Prompt Generation: Update podcastGenerator.ts to accept a format configuration parameter.
  2. Build Sidebar Toggle: Add custom Solo/Dialogue format state and dropdown select into StudioSidebar.tsx.
  3. Adapt TTS Pipeline: Modify podcastAudioGenerator.ts to handle single-speaker direct compilation.
  4. Agent Verification: Run a test script using the paired agent API to compile and verify continuous wav output.
